# SmartFork v2 — Fork Priming (Layer 8)

> **Design:** Generate context files that prime new AI sessions with knowledge from past sessions.
> **4 fork intents:** CONTINUE, REFERENCE, DEBUG, SYNTHESIZE
> **Output:** `/fork.md` files + clipboard support + MCP delivery

---

## ForkAssembler

**File:** `src/smartfork/fork/assembler.py`

```python
class ForkAssembler:
    def __init__(self, llm: LLMProvider, chroma_client,
                 metadata_store: MetadataStore): ...
    
    def assemble(self, session_id: str, intent: ForkIntent,
                 user_query: str = "", max_tokens: int = 2000,
                 use_llm: bool = True) -> ContextReport: ...
```

**Two modes:**
1. **LLM mode (default):** Distills session into narrative context report
2. **Raw fallback:** Structured data assembly without LLM (when Ollama unavailable)

---

## Fork Intent Profiles

Each intent assembles different context layers:

| Intent | Prompt sections | Layers used |
|--------|----------------|-------------|
| CONTINUE | What was built, Approach, Completed, What remains, Open issues, Files | A+B+C+D+F |
| REFERENCE | Problem solved, Approach chosen, Alternatives considered, Key patterns, Gotchas | A+B+C |
| DEBUG | Error encountered, Root cause, What was tried, Fix applied, Prevention | A+E+D |
| SYNTHESIZE | Overview, Timeline of execution, Architectural shifts, Current state, Key files | A+B+C |

---

## Context Layers

Built from ChromaDB + SQLite before LLM call:

| Layer | Content | Source |
|-------|---------|--------|
| A | Task + project + domains + languages + files | SessionDocument |
| B | LLM-generated 3-sentence summary | SessionDocument.summary_doc |
| C | Key decisions (filtered reasoning with decision keywords: "decided", "chose", "approach") | reasoning_docs |
| D | Files touched (edited + read, top 15) | SessionDocument |
| E | Error context (reasoning blocks with error keywords: "error", "bug", "traceback") | reasoning_docs |
| F | Reasoning trail (all reasoning blocks, for CONTINUE) | reasoning_docs |

## Compaction Gap Analysis

When a session was compacted during its lifetime:

```python
def analyze_gaps(self, session_id: str) -> str:
    """Compare compacted summary vs pre-compaction transcript."""
```

1. Check if pre-compaction export exists for this session
2. If yes: agent compares compacted summary ↔ full pre-compaction transcript
3. Identifies: missing error messages, lost decisions, incomplete context
4. Synthesizes: complete context report with gaps filled
5. Adds warning to report: "⚠ This session was compacted. Some context has been reconstructed."

## Share/Export

```python
class ForkExporter:
    def save_to_file(self, report: ContextReport, 
                     output_path: Path = None) -> Path: ...
    def copy_to_clipboard(self, report: ContextReport) -> None: ...
    def deliver_via_mcp(self, report: ContextReport) -> dict: ...
```

- Default filename: `fork_{session_id[:8]}_{intent}.md`
- `--clipboard` flag copies to system clipboard
- `--output PATH` for custom path
- MCP server delivers as tool response
